## The problem

`currentBeliefs`, `causalModel`, `confusions`, `fragileUnderstanding`, `hasExample`, `hasExplanation`, `hasCausalLink` are all generated by the same LLM call that writes the reply. The model simultaneously performs and reports on what it believes. There is no validation that these fields are accurate, stable, or actually derived from what the student said. The state is plausible-looking, not accurate.

## Where it matters most (ranked by severity)

1. **Completion flags** (`hasExample`, `hasExplanation`, `hasCausalLink`) — directly control SUMMARIZE_AND_CLOSE. LLM generosity or one dense student message could set all three after two turns, closing prematurely.
2. **`currentBeliefs` fed into MAKE_PLAUSIBLE_MISTAKE** — if beliefs are hallucinated rather than derived, Pupil makes a mistake about something it invented, not something it learned. Student may not recognise what's being tested.
3. **`fragileUnderstanding` and the move selector** — if hallucinated, wrong move fires. Less catastrophic than 1 and 2 but noisy.

## Root cause

State and reply are entangled in one generation. An LLM good at sounding like it's learning will produce state that *looks* learned. These are correlated abilities, not independent ones.

## Options considered

**A — Monotonic flags + turn floor**
Flags can only go true (never back to false), enforced in `buildMeaningModel`. Minimum `studentClaims.length` before any flag can be set. Light, no latency cost, doesn't fix accuracy but prevents catastrophic early closure.

**B — Separate extraction pass**
Two LLM calls per turn: first extracts/validates state from student's message (constrained JSON-only job, no reply), second generates reply given validated state. Architecturally cleanest. Adds latency and cost.

**C — Ground signals in history**
Replace LLM-reported flags with deterministic checks on actual conversation text. E.g. `hasCausalLink` only true if student's message contains "because", "causes", "leads to", etc. Stable, can't be hallucinated, requires calibration per subject.

**D — currentBeliefs anchored to student quotes**
Post-generation check rejects beliefs that share no vocabulary with what the student actually said. Prevents Pupil from making mistakes about invented beliefs.

## Verdict (not yet acted on)
Option A is minimum viable. Option C is the structural fix for flags. Option B is the right long-term answer for currentBeliefs/fragileUnderstanding. Two-pass architecture (B) matches how a human observer would work — one job to observe, one job to respond — and is the cleanest long-term direction.
